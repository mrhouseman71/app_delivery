Objetivo y Requisitos

Sistema de análisis vocal en tiempo real sobre S.O.S de Bogdan Shuvalov — canción con rango A2–C6 (~3.28 oct.). Detección frame a frame con torchcrepe + pYIN.

Campo note en realtime_frames.csv. Algoritmo torchcrepe (CNN) + pYIN. ~35 notas únicas. Confianza media 0.87.
0.00s→A344.5s→D360.0s→C6
Campo hz. Rango: 109 Hz (A2) → 1059 Hz (C6). Valor continuo por frame.
A2=109HzA4=440HzC6=1059Hz
Campo time_s. 1801 frames 0.00s → 90.00s continuo. Log imprime timestamp por frame.
Campo intensity = RMS×100 via librosa.feature.rms. Pico: 69.35s→C6→36.5. Top 5%: 4 momentos.
Campo register. 5 categorías por umbral Hz. Log: 🟥 SOBREAGUDO@61.5s

Limitaciones y Decisiones

Audio sintético: YouTube bloqueó la descarga (403). Se generó audio programático con armónicos, vibrato y ADSR real de S.O.S.

torchcrepe vs CREPE: TensorFlow no disponible. Misma arquitectura CNN, resultados equivalentes.

Umbral 0.35: conservador para no perder notas cortas de transición.

RMS ≠ proyección vocal: energía acústica objetiva, no percepción subjetiva.
